# 프로젝트 방향 수정 배경

## 배경

본 프로젝트의 초기 목표는 뉴스 전반에서 주요 키워드를 추출하여 트렌드를 파악하는 것이었습니다.
이를 위해 네이버 뉴스 검색 API를 활용하여 다양한 키워드로 데이터를 수집하고, 전체 뉴스 흐름을 반영하는 키워드 통계를 생성하고자 했습니다.

---

## 기존 접근 방식의 한계

### 1. 검색 API 구조로 인한 데이터 편향

네이버 뉴스 API는 "전체 뉴스 데이터"가 아니라 검색어 기반 결과를 반환하는 API입니다.

- 입력한 query에 포함된 기사만 수집되며
- 결과 자체가 이미 필터링된 상태입니다.

이로 인해 "뉴스 전체 트렌드"가 아니라 "특정 검색어 주변의 편향된 데이터"를 분석하게 되는 문제가 발생합니다.

### 2. 본문 데이터 부재

API에서 제공하는 데이터는 `title`, `description`(요약/발췌문) 위주이며, 기사 전체 본문이 포함되지 않습니다.

그 결과:

- 키워드 빈도 분석 시 정보 손실 발생
- 특정 문장만 과대표집
- 일반적인 단어가 상위에 노출되는 문제 발생

### 3. "전체 뉴스 트렌드" 재현의 어려움

초기에는 다양한 키워드를 사용하여 전체 뉴스를 커버하려는 시도를 했습니다.

```
경제, 정치, 사회, 국제, 기술, 기업, 금융, 증시, 부동산, AI
```

하지만 이 방식은:

- 키워드 선택 자체가 또 다른 편향을 만들고
- 도메인 간 불균형 발생 (예: 사회 뉴스 과다)
- 완전한 커버리지 확보 불가능

즉, "전체 뉴스 기반 키워드 통계"는 구조적으로 재현이 어렵다는 결론에 도달했습니다.

---

## 방향 전환: 테마 기반 키워드 분석

### 새로운 목표

특정 도메인(테마)을 정의하고, 해당 도메인 내에서의 키워드 트렌드를 분석한다.

### 핵심 전략

#### 1. 도메인별 키워드 집합 (Keyword Pool) 정의

단일 키워드가 아니라, 도메인을 대표하는 키워드 집합을 기반으로 뉴스 데이터를 수집합니다.

| 도메인 | 키워드 집합 |
|---|---|
| AI / 기술 | AI, 인공지능, 생성형AI, GPT, LLM, 챗GPT, 머신러닝, 딥러닝 |
| 반도체 / 하드웨어 | 반도체, 파운드리, 메모리, GPU, 삼성전자, SK하이닉스, EUV |
| 금융 / 투자 | 금융, 금리, 인플레이션, 환율, 투자, 자산, 은행, 채권 |
| 증시 / 시장 | 증시, 주식, 코스피, 코스닥, 공매도, IPO, 시가총액 |
| 부동산 | 부동산, 아파트, 분양, 전세, 매매, 재건축, 금리 |
| 기업 / 산업 | 기업, 실적, 매출, 영업이익, 투자, M&A, 스타트업 |
| 국제 / 글로벌 | 미국, 중국, EU, 글로벌, 수출, 외교, 무역, 제재 |

#### 2. "편향"을 "의도된 필터"로 전환

- 기존: 검색어 기반 수집 → 문제 (편향 발생)
- 변경: 도메인 기반 수집 → 전략 (필터링 역할)

"전체 뉴스에서 키워드 추출"이 아니라 "특정 도메인 내에서 의미 있는 키워드 추출"로 문제를 재정의합니다.

#### 3. 데이터 품질 개선

테마 기반 접근의 효과:

- 불필요한 뉴스 제거 (노이즈 감소)
- 키워드 의미 강화
- 분석 결과 해석 가능성 증가

---

## 기대 효과

**기존 방식**

```
정부, 발표, 사건, 조사, 관계자...
```

범용 단어 중심으로 인사이트가 부족합니다.

**도메인 기반 방식 — AI 테마 예시**

```
GPT, 생성형AI, GPU, 데이터센터, 반도체, 오픈AI, 모델
```

실제 트렌드 및 산업 변화를 반영합니다.

---

## 확장 방향

- 도메인 간 키워드 비교 분석
- 시계열 기반 트렌드 변화 탐지
- 산업별 이슈 탐지
- 기업/기술 중심 키워드 네트워크 분석

---

## 최종 정리

본 프로젝트는 "전체 뉴스 키워드 분석"에서 "도메인(테마) 기반 뉴스 트렌드 분석 시스템"으로 방향을 전환한다.

이 접근은 데이터 수집 구조의 한계를 해결하면서, 실제 활용 가능한 인사이트를 제공하는 방향이다.
