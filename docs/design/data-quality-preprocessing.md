# 뉴스 트렌드 파이프라인 데이터 품질 전처리 가이드

이 문서는 `news-trend-pipeline-dev`의 실제 처리 흐름을 기준으로, 뉴스 원문이 어떻게 **분석 가능한 키워드 토큰**으로 바뀌는지 쉽게 설명한다. 이 프로젝트의 전처리 결과는 Spark 집계에 바로 사용되는 `list[str]` 형태의 `tokens`다.

---

## 1. 전체 흐름: 뉴스 문장에서 트렌드 키워드까지

```text
뉴스 수집
  ↓
title + summary 결합
  ↓
clean_text()
  ↓
Kiwi 형태소 분석
  ↓
명사만 선택: NNG, NNP
  ↓
사용자 복합명사 사전 적용
  ↓
사용자 불용어 사전 제거
  ↓
tokens 생성
  ↓
keywords / keyword_trends / keyword_relations 집계
```

쉽게 말하면, 전처리는 뉴스 문장을 그대로 저장하는 작업이 아니라 **트렌드 분석에 필요한 단어만 남기는 필터링 과정**이다.

---

## 2. clean_text(): 지저분한 원문을 먼저 정리한다

뉴스 원문에는 HTML 태그, URL, 특수문자, 영문, 숫자, 깨진 문자열이 섞일 수 있다. `clean_text()`는 먼저 이런 노이즈를 제거한다.

| 처리 | 예시 |
|---|---|
| Unicode NFC 정규화 | 글자 조합을 표준 형태로 통일 |
| URL 제거 | `https://news.example.com` 제거 |
| HTML 제거 | `<b>속보</b>` → `속보` |
| 잔여 문자열 제거 | `[+1500 chars]` 제거 |
| 한글 외 문자 제거 | 숫자, 영문, 특수문자 제거 |
| 공백 정리 | 여러 공백을 하나로 정리 |

### 예문

```text
입력:
[속보] <b>AI</b> 반도체 2026년 투자 확대!!! https://news.example.com

clean_text 후:
반도체 투자 확대
```

주의할 점은 현재 구현상 영문과 숫자가 제거된다는 것이다. 그래서 `AI`, `2026` 같은 표현은 그대로는 남지 않는다. 이 규칙은 데이터 품질을 단순하게 유지하는 장점이 있지만, 영문 약어가 중요한 도메인에서는 별도 정책이 필요하다.

---

## 3. 형태소 분석: 문장을 단어 조각으로 나눈다

한국어는 띄어쓰기만으로 단어를 정확히 나누기 어렵다. 예를 들어 `삼성전자는 반도체 투자를 확대했다`를 공백 기준으로만 나누면 `삼성전자는`처럼 조사가 붙은 단어가 그대로 남는다.

이 프로젝트는 Kiwi 형태소 분석기를 사용해 문장을 형태소 단위로 나누고, 그중 명사만 추출한다.

```text
입력:
삼성전자는 반도체 투자를 확대했다

공백 기준 분리:
삼성전자는 / 반도체 / 투자를 / 확대했다

Kiwi 형태소 분석 후 명사만 추출:
삼성전자 / 반도체 / 투자
```

### 왜 형태소 분석이 중요한가?

뉴스 트렌드 분석에서 필요한 것은 문장의 모든 단어가 아니라 **주제어**다. `했다`, `한다`, `위해`, `통해` 같은 표현보다 `삼성전자`, `반도체`, `투자` 같은 명사가 트렌드 집계에 더 유용하다.

현재 구현에서는 Kiwi 결과 중 `NNG` 일반명사와 `NNP` 고유명사만 남긴다.

---

## 4. 사용자 복합명사 사전: 쪼개진 단어를 다시 붙인다

형태소 분석은 유용하지만, 어떤 단어는 너무 잘게 쪼개질 수 있다. 예를 들어 `기준금리`, `인공지능반도체`, `생성형인공지능` 같은 단어는 하나의 개념으로 보는 것이 좋다.

이 프로젝트는 `compound_noun_dict`에 승인된 복합명사를 저장하고, 전처리 시 도메인별 사용자 사전을 Kiwi에 주입한다. 이후 `merge_compound_nouns()`가 토큰 배열을 다시 확인해 복합명사를 병합한다.

### 예문

```text
입력:
한국은행은 기준 금리를 동결했다

형태소 분석 결과:
한국 / 은행 / 기준 / 금리 / 동결

compound_noun_dict에 등록된 단어:
한국은행, 기준금리

복합명사 병합 후:
한국은행 / 기준금리 / 동결
```

### 사용자 사전이 없으면 어떻게 되나?

```text
사용자 사전 없음:
한국 / 은행 / 기준 / 금리 / 동결

사용자 사전 있음:
한국은행 / 기준금리 / 동결
```

즉, 사용자 사전은 모델이 모르는 업무 용어, 산업 용어, 뉴스 용어를 **하나의 의미 단위**로 보존하기 위한 장치다.

---

## 5. 사용자 불용어 사전: 남기면 방해되는 단어를 제거한다

불용어는 자주 나오지만 트렌드 분석에는 큰 도움이 되지 않는 단어다. 이 프로젝트는 `stopword_dict`에서 도메인별 불용어를 조회하고, 토큰이 불용어에 포함되면 제거한다.

DB를 사용할 수 없는 환경에서는 기본 불용어 목록을 fallback으로 사용한다. 예를 들어 `기자`, `뉴스`, `이번`, `지난`, `관련`, `위해`, `통해`, `올해`, `시장`, `기술`, `발표`, `계획` 같은 단어가 기본 불용어에 포함되어 있다.

### 예문

```text
입력:
삼성전자는 올해 반도체 투자 계획을 발표했다고 밝혔다

형태소 분석 + 명사 추출:
삼성전자 / 올해 / 반도체 / 투자 / 계획 / 발표

stopword_dict 예시:
올해, 계획, 발표

불용어 제거 후:
삼성전자 / 반도체 / 투자
```

### 왜 사용자 불용어 사전이 필요한가?

모든 도메인에서 같은 단어가 항상 불용어인 것은 아니다.

```text
경제 도메인:
시장 → 너무 자주 나와서 불용어일 수 있음

IT 도메인:
시장 → "AI 시장", "반도체 시장"처럼 중요한 분석어일 수 있음
```

그래서 이 프로젝트는 domain별 사전을 사용한다. 공통 불용어는 `all`에 둘 수 있고, 특정 도메인에만 불필요한 단어는 해당 domain에만 등록한다.

---

## 6. 한 문장이 실제로 바뀌는 과정

```text
원문:
[속보] 삼성전자는 올해 AI 반도체 시장 투자 확대 계획을 발표했다고 밝혔다. <br> 김OO 기자
```

현재 `clean_text()` 규칙은 영문과 숫자를 제거하므로 다음처럼 정리된다.

```text
1) clean_text 후:
삼성전자는 올해 반도체 시장 투자 확대 계획을 발표했다고 밝혔다 김 기자
```

Kiwi 형태소 분석과 명사 필터를 거치면 조사와 동사가 줄어든다.

```text
2) 형태소 분석 + 명사 추출:
삼성전자 / 올해 / 반도체 / 시장 / 투자 / 확대 / 계획 / 발표 / 김 / 기자
```

복합명사 사전에 `반도체시장`이 등록되어 있다면 의미 단위로 병합된다.

```text
3) 복합명사 병합:
삼성전자 / 올해 / 반도체시장 / 투자 / 확대 / 계획 / 발표 / 김 / 기자
```

불용어 사전에 `올해`, `확대`, `계획`, `발표`, `기자`가 등록되어 있다면 제거된다.

```text
4) 불용어 제거 후 최종 tokens:
삼성전자 / 반도체시장 / 투자
```

최종적으로 Spark 집계에는 `삼성전자`, `반도체시장`, `투자`처럼 트렌드 분석에 더 적합한 토큰이 사용된다.

---

## 7. 사전은 어떻게 좋아지나?

프로젝트는 전처리 사전을 사람이 한 번 만들고 끝내는 구조가 아니다. 데이터가 쌓이면 사전 후보를 추천하고, 승인된 단어를 다시 전처리에 반영한다.

### 복합명사 후보 흐름

```text
news_raw(title, summary, domain)
  ↓
사용자 사전 없이 Kiwi 분석
  ↓
연속된 명사 조합 탐색
  ↓
compound_noun_candidates 저장
  ↓
자동 평가 또는 관리자 승인
  ↓
compound_noun_dict 반영
  ↓
다음 전처리부터 복합명사로 병합
```

후보 추출 단계에서 사용자 사전을 일부러 주입하지 않는 이유는, 이미 아는 단어가 아니라 **아직 사전에 없는 새 복합명사 후보**를 찾기 위해서다.

### 불용어 후보 흐름

```text
keyword_trends / keywords / keyword_relations
  ↓
너무 넓게 반복되는 단어 탐지
  ↓
stopword_candidates 추천
  ↓
관리자 검토
  ↓
stopword_dict 반영
  ↓
다음 전처리부터 제거
```

예를 들어 `관련`, `이번`, `기자`처럼 여러 도메인에서 반복되지만 분석 의미가 약한 단어는 불용어 후보가 될 수 있다.

---

## 8. 데이터 품질을 결정하는 핵심 기준

| 기준 | 나쁜 결과 | 좋은 결과 |
|---|---|---|
| 정제 | URL, HTML, 특수문자가 토큰에 섞임 | 한글 중심 본문만 남음 |
| 형태소 분석 | `삼성전자는`, `투자를`처럼 조사가 붙음 | `삼성전자`, `투자`처럼 명사만 남음 |
| 복합명사 | `기준 / 금리`, `한국 / 은행`으로 쪼개짐 | `기준금리`, `한국은행`으로 의미 보존 |
| 불용어 | `기자`, `이번`, `관련`이 상위 키워드로 노출 | 실제 이슈 키워드가 상위에 노출 |
| 도메인 사전 | 모든 도메인에 같은 규칙 적용 | 도메인별 중요 단어와 불용어를 다르게 관리 |

---

## 9. 한 줄 정리

```text
좋은 전처리 = 원문 노이즈 제거 + 형태소 분석으로 명사 추출 + 복합명사 보존 + 사용자 불용어 제거
```

이 흐름이 잘 동작해야 `keywords`, `keyword_trends`, `keyword_relations`의 품질이 좋아지고, 대시보드에서 보이는 트렌드도 실제 뉴스 이슈에 가까워진다.
